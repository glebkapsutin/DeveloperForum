# **DevelopForum**

## **Описание проекта**

**DevelopForum** — это форум для разработчиков, где они могут делиться своими знаниями, задавать вопросы и обсуждать темы, связанные с программированием. Проект разработан как демонстрация навыков в **C#**, **ASP.NET Core**, **React.js** и работе с базами данных.  

Цель проекта — создать полноценное веб-приложение, которое отображает глубокие знания в разработке, проектировании и применении чистой архитектуры.

---

## **Технологии**

### **Backend**:
- **Язык**: C#
- **Фреймворк**: ASP.NET Core
- **База данных**: PostgreSQL
- **ORM**: Entity Framework Core

### **Frontend**:
- **Язык**: JavaScript (TypeScript при необходимости)
- **Фреймворк**: React.js
- **Управление состоянием**: Redux или Context API
- **Стилизация**: CSS/SCSS или библиотеки компонентов (например, Material-UI)

### **Инструменты и среды разработки**:
- **IDE**: Visual Studio (Windows) / VS Code (MacOS)
- **Система контроля версий**: Git (GitHub для репозитория)
- **Система миграций**: EF Core Migrations
- **Пакетный менеджер**: npm или yarn

## **Функциональные блоки**

### **1. Аутентификация и авторизация**
- Регистрация новых пользователей.
- Вход в систему (логин).
- Подтверждение e-mail (опционально).
- Сброс пароля.
- Роли пользователей:
  - **Обычный пользователь**: создание постов, комментирование, голосование.
  - **Модератор**: управление постами и комментариями.
  - **Администратор**: управление пользователями, их ролями и правами.

### **2. Форум**
- **Создание постов**: Пользователи могут добавлять тематические посты.
- **Список постов**: Сортировка постов по дате создания, популярности, или категории.
- **Просмотр поста**: Отображение содержимого, комментариев и голосов.
- **Комментарии**: Возможность добавлять и удалять комментарии.
- **Рейтинг постов и комментариев**: Лайки/дизлайки для оценки содержания.

### **3. Пользовательский профиль**
- Просмотр информации о пользователе (имя, фото, дата регистрации, активность).
- Редактирование профиля (изменение имени, аватара, пароля).
- Просмотр списка созданных постов и комментариев.

### **4. Административные функции**
- Управление пользователями (активация/деактивация аккаунтов, назначение ролей).
- Модерация контента (удаление постов и комментариев, блокировка пользователей).
- Управление категориями (создание/удаление категорий для постов).

### **5. Уведомления**
- Уведомления о новых комментариях к постам.
- Уведомления о лайках или упоминаниях пользователя.

## **Архитектура**

Приложение построено на основе **чистой архитектуры (Clean Architecture)** и разделено на несколько слоёв:

### **1. Core**
- **Models**: Основные модели данных (например, `User`, `Post`, `Comment`, `Vote`).
- **Enums**: Перечисления для ролей, статусов и других атрибутов.
- **DTOs**: Объекты передачи данных для взаимодействия между слоями.

### **2. Application**
- **Interfaces**: Интерфейсы для реализации сервисов и репозиториев.
- **Services**: Бизнес-логика (например, `AuthService`, `PostService`, `CommentService`).

### **3. Infrastructure**
- **Data**: Контекст базы данных (`DbContext`) и миграции.
- **Repositories**: Реализация репозиториев для работы с базой данных.

### **4. Presentation**
- **Controllers**: API-контроллеры для обработки запросов.
- **ViewModels**: Модели для передачи данных в интерфейс.

## **План разработки**

1. **Начальная настройка проекта**:
   - Создать ASP.NET Core проект.
   - Настроить Entity Framework и подключение к базе данных.
   - Настроить React.js проект и интеграцию с бэкендом.

2. **Реализация базовой структуры**:
   - Создать модели данных (`User`, `Post`, `Comment`, `Vote`).
   - Настроить репозитории для работы с базой данных.

3. **Реализация функциональности**:
   - Аутентификация и авторизация (JWT-токены, роли).
   - CRUD-операции для постов и комментариев.
   - Реализация голосования за посты.

4. **Интерфейс пользователя**:
   - Разработать страницы для отображения постов и комментариев.
   - Реализовать функционал создания и редактирования контента.

5. **Административная панель**:
   - Реализовать управление пользователями и контентом.

6. **Тестирование и оптимизация**:
   - Написать юнит-тесты для ключевых компонентов.
   - Оптимизировать работу базы данных.

7. **Деплой**:
   - Настроить CI/CD.
   - Развернуть приложение на сервере (Heroku, AWS, Azure).

## **Как развернуть проект локально**

1. Клонируйте репозиторий:
   ```bash
		https://github.com/glebkapsutin/DeveloperForum.git
   ```
2. Перейдите в папку с проектом:
   ```bash
   cd developforum
   ```
3. Настройте базу данных в файле `appsettings.json`.

4. Выполните миграции для базы данных:
   ```bash
   dotnet ef database update
   ```

5. Запустите проект:
   ```bash
   dotnet run
   ```

6. Для запуска фронтенда перейдите в папку `client` и установите зависимости:
   ```bash
   npm install
   ```
   Затем запустите фронтенд:
   ```bash
   npm start
   ```
